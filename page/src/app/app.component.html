Decentralised software development on GitHub An open source browser extension for the decentralized management of GitHub-Repositories on the ethereum blockchain.

deVote functionalities
Crowdfund issue solutions
The community can crowdfund a bounty for the solving of an issue.
Solve issues for a bounty
A developer can solve an issue for a bounty if his solution is accepted by the community.
Vote on pull-requests
The can community votes whether a pull-request should be merged or not.
Advantages of using deVote
deVote
Community driven
The community decides which issues are important and need to be addressed first.
Financial initiative
Developers have a financial initiative to solve issues because of the Bounty.
Qualitative merges
Many people can participate in the voting and thus better validate whether a pull request meets the quality standard.
Traditional
Admin dependent
The admin decides which issue comes first and can therefore hold back development.
Free developer dependent
The repository must hope that developers will continue to develop the project in their spare time and for free.
Pull-Requests bottleneck
It doesn't matter what the community thinks about a pull request, in the end it's the admin alone who decides whether to merge it or not.
